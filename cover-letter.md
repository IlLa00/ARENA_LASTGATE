# 나인비 스튜디오 지원 자기소개서

## 지원동기

저는 오픈월드 게임을 만들고 싶어서 프로그래머가 됐습니다.

정보통신 분야에서 안정적으로 일하던 중, 오픈월드 게임을 직접 만들고 싶다는 생각이 떠나지 않았습니다. 결국 직장을 그만두고 게임 프로그래밍을 처음부터 공부했습니다. 단순히 취업을 위한 선택이 아니라, 오픈월드라는 장르 자체에 매료된 게이머이자 그 안에 들어가는 시스템을 직접 설계하고 싶은 프로그래머로서의 결정이었습니다.

Project Orion을 알게 됐을 때, 제가 찾던 프로젝트라는 확신이 들었습니다. 제가 가장 좋아하는 게임 장르인 오픈월드 서바이벌이고, GAS 기반 개발 경험이 요구사항과 정확히 맞아 떨어졌습니다. 기술 스택도, 프로젝트 방향도, 제가 지금까지 쌓아온 것들이 이 프로젝트를 위한 것이었나 싶을 정도였습니다. 뼈를 묻을 수 있는 프로젝트를 찾고 있었고, Project Orion이 그 답이라고 생각합니다.

## 언리얼 엔진 5 — GAS와 멀티플레이 서버 경험

팀 프로젝트 **ARENA LASTGATE**에서 프로그래머 팀장을 맡아 UE5.5 기반 4인 멀티플레이 배틀로얄 게임을 3개월간 개발했습니다. 제가 직접 담당한 세 가지 시스템을 통해 역량을 설명드리겠습니다.

**GAS 기반 무기 개조 시스템**

Custom MMC로 `BaseDamage * (1 + DamageMulti) * (1 + DamageAmpMulti)` 수식을 구현하고, SetByCaller로 DataTable 수치를 GameplayEffect에 동적 주입했습니다. GameWorldSubsystem에서 DataTable을 초기화 시점에 캐싱하고 TSoftObjectPtr을 UPROPERTY TMap에 강참조로 유지해 GC 이슈를 사전에 차단했습니다.

**서버 권한 상점 및 인벤토리 시스템**

클라이언트 요청은 PlayerController Server RPC로 진입하고, 실제 거래 로직은 ShopManager에서 서버 전용으로 처리합니다. 재화 차감은 AddItem 성공 이후에만 실행되도록 순서를 설계했고, ItemSlot을 경량 구조체로 정의해 DOREPLIFETIME으로 복제하고 OnRep 콜백으로 UI를 갱신했습니다.

**LineTrace 기반 실시간 시야 시스템**

원뿔 시야를 다수의 LineTrace로 계산하고, 충돌 결과로 삼각형을 생성해 RenderTarget에 직접 드로우했습니다. 섬광탄 피격 시 RenderTarget을 블랙으로 클리어하고 플레이어 위젯을 Hidden 처리하는 예외 흐름도 전담했습니다. CPUPROFILER 매크로를 삽입해 프레임 단위 성능을 직접 모니터링하며 최적화했습니다.

## C++ 엔진 구조 직접 설계 — Project SANNABI

산나비 모작 포트폴리오를 WinAPI와 C++로 1달간 혼자 제작했습니다. 외부 엔진 없이 Actor-Component 구조를 직접 설계하고, 함수 포인터 기반의 이벤트 시스템(OnComponentBeginOverlap / OnComponentEndOverlap)을 구현했습니다.

물리 시스템은 EPhysicsState 상태 머신으로 Normal / Grappling / WallClimbing / CeilingHang을 분리했고, 그래플 스윙은 `각가속도 = (g / L) * cos(θ)` 진자 공식 기반으로 구현해 입력 타이밍에 따른 관성 보너스까지 반영했습니다. 충돌은 AABB + 법선 벡터 계산으로 지면/천장/좌우 벽을 구분하고, 채널 필터링 매트릭스로 충돌 쌍 규칙을 관리했습니다.

이 포트폴리오를 통해 엔진이 내부적으로 어떻게 동작하는지를 직접 구현해보며 이해했고, 그 경험이 UE5에서 시스템을 설계할 때 근거 있는 판단을 내리는 데 도움이 됩니다.

## AI를 도구로 다루는 개발자

저는 Claude Code를 개발 워크플로우에 적극적으로 통합하고 있습니다. 현재 오픈월드 기술 포트폴리오(동적 날씨 변화, 렌더링 최적화)를 개발하면서 Claude Code로 빠른 출시를 목표로 활용 중입니다. 팀 프로젝트에서는 AI를 통한 코드리뷰 방식을 동료들과 공유했는데, "그렇게 생각할 수도 있구나"라는 반응처럼 새로운 시각을 전달하는 경험을 했습니다.

AI 시대를 이해하기 위해 『AI 2026』과 『AI시대의 질문력』을 읽고 있습니다. 기술이 빠르게 변하는 시대일수록 AI를 경쟁자가 아닌 확장 도구로 삼아, 설계와 판단에 집중하는 프로그래머가 살아남는다고 생각합니다.

## 입사 후 포부

Project Orion의 핵심 개발진이 되고 싶습니다.

GAS 기반 시스템 설계, 멀티플레이 서버 구조, 오픈월드 기술 요소까지 — 제가 지금까지 집중해온 것들이 Project Orion이 필요로 하는 것과 일치합니다. 빠르게 팀에 기여하고, 장기적으로는 프로젝트의 기술 방향을 함께 고민할 수 있는 아키텍트로 성장하고 싶습니다. 급변하는 AI 시대에 프로그래머의 역할은 구현자가 아닌 설계자로 진화해야 한다고 믿으며, 그 방향으로 꾸준히 성장해 나가겠습니다.

오픈월드를 만들고 싶어서 시작한 길의 끝이 Project Orion이길 바랍니다.
