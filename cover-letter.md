## 지원동기

저는 오픈월드 게임을 만들고 싶어서 프로그래머가 됐습니다.

정보통신 분야에서 일하던 시절, 저는 늘 성장에 대한 고민을 안고 살았습니다. 안정적인 환경이었지만, 지금 이 자리에 10년 뒤에도 서 있을 수 있을까라는 질문이 계속 따라붙었습니다. 그 답을 찾던 중 오픈월드 게임을 마주했고, 처음으로 "이걸 내 손으로 만들고 싶다"는 확신이 생겼습니다. 드넓은 세계를 탐험하는 게이머로서 느꼈던 그 감정을 직접 설계하고 싶었습니다. 결국 성장하지 못하는 현재보다 가능성이 있는 미래를 선택했고, 퇴사 후 게임 프로그래밍을 처음부터 공부했습니다.

Project Orion을 알게 됐을 때, 제가 찾던 프로젝트라는 확신이 들었습니다. 제가 가장 좋아하는 게임 장르인 오픈월드 서바이벌이고, GAS 기반 개발 경험이 요구사항과 정확히 맞아 떨어졌습니다. 기술 스택도, 프로젝트 방향도, 제가 지금까지 쌓아온 것들이 이 프로젝트를 위한 것이었나 싶을 정도였습니다. 뼈를 묻을 수 있는 프로젝트를 찾고 있었고, Project Orion이 그 답이라고 생각합니다.

## 언리얼 엔진 5 — GAS와 멀티플레이 서버 경험

팀 프로젝트 **ARENA LASTGATE**에서 프로그래머 팀장을 맡아 UE5.5 기반 4인 멀티플레이 배틀로얄 게임을 3개월간 개발했습니다. 제가 직접 담당한 세 가지 시스템을 통해 역량을 설명드리겠습니다.

**GAS 기반 무기 개조 시스템**

언리얼의 GAS(Gameplay Ability System)는 캐릭터의 능력치와 스킬을 데이터 기반으로 관리하는 프레임워크입니다. 저는 이 시스템 위에 무기 개조 기능을 직접 설계했습니다. 개조 아이템마다 기본 데미지, 데미지 배율, 증폭 배율을 독립적으로 조합할 수 있도록 수식을 정의하고, 개조 데이터는 외부 테이블에서 읽어와 런타임에 동적으로 주입했습니다. 덕분에 기획 수치가 바뀌어도 코드를 건드리지 않고 테이블만 수정하면 되는 구조를 만들었습니다. 또한 게임 시작 시점에 데이터를 메모리에 올려두고 GC(가비지 컬렉터)가 임의로 해제하지 못하도록 강참조로 고정해, 런타임 로딩 없이 즉시 접근할 수 있게 했습니다.

**서버 권한 상점 및 인벤토리 시스템**

멀티플레이 게임에서 재화와 아이템 처리를 클라이언트에 맡기면 핵이나 패킷 조작에 취약합니다. 이를 방지하기 위해 클라이언트는 "구매하고 싶다"는 요청만 서버로 보내고, 실제 재화 차감과 아이템 지급은 서버에서만 처리하도록 권한을 분리했습니다. 재화 부족, 슬롯 부족, 수량 초과 등 모든 유효성 검증도 서버에서 수행하며, 아이템 추가가 실제로 성공한 이후에만 재화를 차감하는 순서를 명확히 했습니다. 아이템 슬롯 데이터는 서버가 변경하면 자동으로 클라이언트에 동기화되어 UI가 갱신됩니다.

**LineTrace 기반 실시간 시야 시스템**

플레이어 주변을 부채꼴 모양으로 수십 개의 광선을 쏘아, 벽이나 장애물에 막히는 지점까지만 시야가 열리도록 계산합니다. 계산된 경계점들을 삼각형으로 이어붙여 텍스처에 직접 그리고, 그 텍스처를 미니맵과 시야 마스크에 활용했습니다. 섬광탄을 맞으면 텍스처를 검정으로 덮어 시야를 완전히 차단하고, 타워를 점령하면 해당 지역의 시야가 추가로 열리는 예외 흐름도 담당했습니다. 성능 모니터링 도구를 직접 삽입해 매 프레임 처리 시간을 측정하며 최적화했습니다.

## C++ 엔진 구조 직접 설계 — Project SANNABI

산나비 모작 포트폴리오를 WinAPI와 C++로 1달간 혼자 제작했습니다. 유니티나 언리얼 같은 외부 엔진을 사용하지 않고, 게임 오브젝트가 여러 기능(물리, 충돌, 렌더링)을 독립적인 부품처럼 조립해서 동작하는 구조를 직접 설계했습니다. 각 부품이 서로 직접 참조하지 않고 이벤트를 통해 통신하도록 만들어, 코드 간 의존성을 낮췄습니다.

물리 시스템은 캐릭터가 처한 상황(일반 이동 / 그래플 스윙 / 벽 타기 / 천장 매달리기)마다 다른 물리 로직이 적용되도록 상태를 명확히 분리했습니다. 그래플 훅 스윙은 실제 진자 운동 공식을 적용해 자연스럽게 움직이도록 구현했고, 플레이어 입력 타이밍에 따라 스윙 폭이 커지는 보너스를 추가해 조작감을 살렸습니다. 충돌은 물체가 어느 면에 부딪혔는지(바닥인지, 천장인지, 왼쪽 벽인지)를 계산해 상태를 전환하고, 어떤 종류의 물체끼리만 충돌을 검사할지 규칙을 표로 관리했습니다.

엔진 없이 직접 구현해본 경험은 언리얼 엔진이 내부적으로 왜 그렇게 동작하는지를 이해하는 데 실질적인 기반이 됐습니다.

## AI를 도구로 다루는 개발자

저는 Claude Code를 개발 워크플로우에 적극적으로 통합하고 있습니다. 현재 오픈월드 기술 포트폴리오(동적 날씨 변화, 렌더링 최적화)를 개발하면서 Claude Code로 빠른 출시를 목표로 활용 중입니다. 팀 프로젝트에서는 AI를 통한 코드리뷰 방식을 동료들과 공유했는데, "그렇게 생각할 수도 있구나"라는 반응처럼 새로운 시각을 전달하는 경험을 했습니다.

AI 시대를 이해하기 위해 『AI 2026』과 『AI시대의 질문력』을 읽고 있습니다. 기술이 빠르게 변하는 시대일수록 AI를 경쟁자가 아닌 확장 도구로 삼아, 설계와 판단에 집중하는 프로그래머가 살아남는다고 생각합니다.

## 입사 후 포부

Project Orion의 핵심 개발진이 되고 싶습니다.

GAS 기반 시스템 설계, 멀티플레이 서버 구조, 오픈월드 기술 요소까지 — 제가 지금까지 집중해온 것들이 Project Orion이 필요로 하는 것과 일치합니다. 빠르게 팀에 기여하고, 장기적으로는 프로젝트의 기술 방향을 함께 고민할 수 있는 아키텍트로 성장하고 싶습니다. 급변하는 AI 시대에 프로그래머의 역할은 구현자가 아닌 설계자로 진화해야 한다고 믿으며, 그 방향으로 꾸준히 성장해 나가겠습니다.

오픈월드를 만들고 싶어서 시작한 길의 끝이 Project Orion이길 바랍니다.
